## Get Started with theutilyes

This guide will help you start using `theutilyes` utility functions in your project.

<Steps>
  <Step title="Install the package">
    First, install `theutilyes` via npm, yarn, or pnpm:
    
    ```bash
    npm install theutilyes
    ```
    
    See the [Installation](/installation) guide for other package managers.
  </Step>

  <Step title="Import the utilities">
    Import the functions you need from the package:
    
    ```typescript
    import { isOdd, isEven, isEqual, getFiles } from 'theutilyes';
    ```
    
    <Tip>
      You can import individual functions or use wildcard imports: `import * as util from 'theutilyes'`
    </Tip>
  </Step>

  <Step title="Use the utilities">
    Start using the utility functions in your code:
    
    ```typescript
    import { isOdd, isEven, isEqual, getFiles } from 'theutilyes';
    
    // Check if numbers are odd or even
    console.log(isOdd(3));   // true
    console.log(isEven(4));  // true
    
    // Compare values
    console.log(isEqual(5, 5));       // true
    console.log(isEqual('a', 'b'));   // false
    
    // Get files from a directory
    const jsFiles = getFiles('./src', '.js', false);
    console.log(jsFiles); // Array of .js files
    ```
  </Step>
</Steps>

## Common Use Cases

### Working with Numbers

```typescript
import { isOdd, isEven, isNotOdd, isNotEven } from 'theutilyes';

const num = 42;

if (isEven(num)) {
  console.log('Number is even');
}

if (isNotOdd(num)) {
  console.log('Number is not odd');
}
```

### File System Operations

```typescript
import { getFiles, isDirectory, isFile } from 'theutilyes';

// Get all TypeScript files from a directory
const tsFiles: string[] = getFiles('./src', '.ts', false);

// Create directory if it doesn't exist
const configFiles: string[] = getFiles('./config', '.json', true);

// Check if path is a directory or file
if (isDirectory('./src')) {
  console.log('It\'s a directory');
}

if (isFile('./package.json')) {
  console.log('It\'s a file');
}
```

### Boolean Conversions

```typescript
import { toBoolean, isTrue, isFalse } from 'theutilyes';

const value = 'hello';

console.log(toBoolean(value));  // true
console.log(isTrue(value));     // true
console.log(isFalse(0));        // true
```

### Discord Emoji Utilities

```typescript
import { isAnimatedEmoji, parseEmoji } from 'theutilyes';

const emoji = '<a:partyparrot:123456789>';

if (isAnimatedEmoji(emoji)) {
  console.log('This is an animated emoji!');
}

const parsed = parseEmoji(emoji);
console.log(parsed); // :partyparrot:
```

<Note>
  The emoji functions expect Discord emoji format: `<a:name:id>` for animated or `<:name:id>` for static emojis.
</Note>

## TypeScript Support

`theutilyes` is written in TypeScript and provides type definitions. Some functions include explicit type annotations:

```typescript
import { getFiles, isH } from 'theutilyes';

// getFiles has full type signature
const files: string[] = getFiles(
  './src',      // directory: string
  '.ts',        // extension: string
  false         // createDirIfNotFound: boolean
);

// isH checks for the number 8 or string 'h'
const result = await isH(8);      // true
const result2 = await isH('h');   // true
const result3 = await isH(7);     // false
```

## Easter Egg

```typescript
import { theField } from 'theutilyes';

// Downloads a special gif to your current directory
await theField();
```

<Tip>
  This function downloads a gif file to your current directory as `the_field.gif`. Use with caution!
</Tip>
